Web Standards: Content in HTML, Visual design in CSS, Functionality & enhancements in JavaScript The Six Rules of HTML:
  1. Every valid HTML document's very first line must be a DOC-TYPE declaration
  2. Every tag that opens around taxt content must close
  3. Tags close in the opposite order that they open
  4. All tag elements and attributes, and some attribute values, must be lowercase letters
  5. All attribute values must appear in quotation marks, immediately following the equals sign
  6. All class and id values should begin with a letter. and must not contain spaces
Global Structure of HTML:
  • html-root element; the tag that contains the entire webpage & should have atleast the "lang" attribute
  • head-tag before the "body" tag that contains metadata, must contain contain two important tags: title-the title of the page that appears in the title bar/tab of web browsers & bookmarks (CH15). meta-tag with a "charset="utf-8"" attribute that informs browsers &search engines of page encoding
  • body-tag contains all the page content that is expected to be visible in a browser's viewport

CSS-Cascading Style Sheets:
  • describe the visual presentation & design of webpages, including typography, color & page layout
  • CSS controls the appearance of all content structure of HTML
  • CSS rules written near the top of the file will be rendered first & rules near the bottom can override identical styles near the top
A list of rules (rulesets) that are comprised of two parts: a selector, which specifies which structural parts of the HTML document will be styled & a declaration block, which uses curly braces to group a list of one or more style properties & their values. Some properties take a single value & others take multiple values. 
selector{
      property: value;
    }
The basic CSS selectors are HTML tags; (Element selectors or Type selectors): 
body {
      background-color: lightgrey;
      color: blue;
    }
    
h1 {
      background-color: black;
      color: white;
    }
Class selectors begin with a dot & ID selectors begin with a hash: 
.warning {
      color: red;
      font-weight: bold;
    }
    
#instructions {
      background-color: black;
      color: white;
    }
Attribute Selectors turn any HTML attribute into a CSS selector with a more complicated syntax
  • [attr] selects any element with the attr attribute: img[alt] will select all image tags with an alt attribute
  • [attr="value"] selects any element with the attr whose value is exactly value:*[lang="de"] will select any HTML element with de (German) as its lang attribute
  • [attr^="value"] selects any element with the attr attribute whose value BEGINS WITH value: 
    a [href^="http"] { 
        color: green 
      }
    will make all external links green
  • [attr$="value"] selects any element with the attr attribute whose value ENDS WITH value: a[href$=".com"] selects all anchor tags with links pointing to .com domains
  • [attr*="value"] selects any element with the attr attribute whose value contains value atleast once: img[src*="cat"] will select any image tag whose src attribute refers to an image file with cat somewhere in the file name or path
Pseudoclass Selectors select an element based on either user behavior or the structure of the HTML document. The more common pseudoclass selectors are used for styling link text appearing inside the a-anchor tag:
  • :link styles a link in its unvisited state
  • :visited styles a link that has been visited
  • :hover styles a link when a mouse pointer is hovering over it (usually combined with focus)
  • :focus styles a link that has been tabbed to by the keyboard
  • :active styles a link during the brief moment that it has been activated by an action (mouse click, tap, Return)
a, a:link {
      color: green;
      text-decoration: none;
    }
    a:hover, a:focus {
      text-decoration: underline;
    }
    a:visited { color: red; }
